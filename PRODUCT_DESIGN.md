# 产品设计文档 - 顶级主厨（TopChef）

## 1. 产品概述

### 1.1 产品名称
顶级主厨（TopChef）

### 1.2 产品定义
一个微信小程序应用，帮助用户记录和管理自己会做的菜肴，包括菜肴信息、制作步骤、食材清单等功能。支持用户与朋友分享菜谱、评论交流、相互关注，形成一个社交化的烹饪知识社区，方便用户随时查阅和分享烹饪经验。

### 1.3 产品目标
- 提供便捷的菜肴记录和管理工具
- 帮助用户管理和组织自己的烹饪知识库
- 支持菜肴搜索和分类查看
- 建立社交分享平台，促进用户之间的菜谱交流
- 提升用户的烹饪体验和效率，增强社区互动

### 1.4 目标用户
- 年龄：18-50岁
- 特征：喜欢烹饪或学习烹饪的用户，对菜谱分享和交流有兴趣
- 需求：需要一个个人菜肴管理工具，同时希望与他人分享和学习烹饪知识

### 1.5 核心价值主张
- **个人化管理**：随时记录和整理自己的拿手菜
- **社交互动**：与朋友分享菜谱，获得反馈和建议
- **知识积累**：浏览其他用户的菜谱，学习新的烹饪技能
- **便利性**：随时随地通过微信小程序访问和查看

## 2. 产品功能需求

### 2.1 核心功能模块

#### 2.1.1 菜肴管理
- **添加菜肴**
  - 支持输入菜名、描述、难度级别、烹饪时间
  - 支持上传菜肴图片
  - 支持设置菜肴标签（如快手菜、家常菜、宴客菜等）
  
- **菜肴详情**
  - 显示基本信息（菜名、图片、难度、时间）
  - 展示食材列表（支持数量和单位）
  - 展示制作步骤（支持文字和图片）
  - 显示标签和收藏状态
  
- **编辑菜肴**
  - 支持修改菜肴信息
  - 支持修改食材和步骤
  
- **删除菜肴**
  - 支持删除菜肴（确认删除）

#### 2.1.2 菜肴浏览
- **菜肴列表**
  - 列表展示所有菜肴（卡片样式）
  - 支持按难度、时间排序
  - 支持按标签筛选
  - 支持搜索菜肴
  - 列表分页加载
  
- **分类查看**
  - 按标签分类展示
  - 按难度分类展示

#### 2.1.3 收藏功能
- 支持收藏喜欢的菜肴
- 单独的收藏列表页面
- 从详情页快速收藏/取消收藏

#### 2.1.4 评论和评分（新增）
- **菜肴评分**
  - 支持 1-5 星评分
  - 显示菜肴平均评分
  - 显示评分分布图表
  
- **菜肴评论**
  - 支持对菜肴添加评论
  - 显示评论列表（最新/热门排序）
  - 支持删除自己的评论
  - 支持评论通知提醒

#### 2.1.5 菜肴分享（新增）
- **分享菜肴**
  - 支持分享给微信朋友
  - 支持分享给微信朋友圈
  - 支持分享菜肴二维码
  - 分享包含菜肴基本信息和链接

- **分享统计**
  - 显示菜肴被分享的次数
  - 显示菜肴的浏览量

#### 2.1.6 个人中心
- 显示用户基本信息（昵称、头像）
- 显示菜肴统计（总数、按难度分布等）
- 显示最近添加的菜肴
- 显示粉丝/关注数统计
- 支持设置和关于

#### 2.1.7 用户互动（新增）
- **用户主页**
  - 显示用户发布的菜肴列表
  - 显示用户基本信息和粉丝数
  - 支持关注/取消关注用户
  
- **关注功能**
  - 支持关注其他用户
  - 支持取消关注
  - 显示粉丝列表和关注列表

- **账户系统**
  - 微信登录/授权
  - 用户基本信息完善
  - 账户设置和隐私设置

#### 2.1.8 菜单管理（新增）
- **创建菜单**
  - 支持创建多个菜单（如：周一菜单、家常菜单、宴客菜单等）
  - 菜单命名和描述
  - 菜单分类标签

- **菜单编辑**
  - 支持从菜肴库中添加菜肴到菜单
  - 支持设置每道菜的份量和服务人数
  - 支持拖拽菜肴调整菜单顺序
  - 支持批量添加菜肴

- **菜单管理**
  - 查看菜单列表
  - 编辑菜单信息
  - 删除菜单
  - 复制菜单快速创建
  - 菜单封面和预览

- **菜单分享**
  - 支持分享菜单给朋友
  - 支持生成菜单购物清单（合并同类食材）
  - 支持菜单评分和评论

#### 2.1.9 点菜系统（新增）
- **快速点菜**
  - 支持"今天吃什么"随机推荐功能
  - 支持按需求点菜（快手菜、健康、家常等分类）
  - 支持根据现有食材推荐菜肴

- **菜肴搭配**
  - 显示菜肴营养搭配建议
  - 显示菜肴口味搭配（如配合推荐）
  - 支持自定义菜单组合

- **购物清单**
  - 根据选中菜肴自动生成购物清单
  - 合并相同食材
  - 支持按超市分类整理食材
  - 支持核对清单功能（打勾标记）
  - 支持分享购物清单给家人

- **菜肴评估**
  - 显示总烹饪时间和难度估计
  - 显示所需食材数量和成本估算
  - 显示营养信息汇总

### 2.2 非功能需求

#### 2.2.1 性能
- 列表加载响应时间 < 300ms
- 搜索响应时间 < 500ms
- 评论列表加载时间 < 500ms
- 图片加载采用懒加载
- API 响应时间 < 1s

#### 2.2.2 用户体验
- 支持离线浏览已加载的菜肴（本地缓存）
- 支持数据本地缓存
- 操作反馈提示（加载、删除等）
- 网络异常友好提示和重试机制
- 云端数据与本地数据的同步

#### 2.2.3 安全性
- 微信登录认证，确保用户身份
- 用户数据加密传输（HTTPS）
- 用户隐私相关数据的访问控制
- 评论和分享内容的审查机制（可选）

#### 2.2.4 兼容性
- 支持 iOS 和 Android 微信
- 适配不同屏幕尺寸
- 支持微信相关权限（相册、位置等）

#### 2.2.5 可用性
- API 可用性 > 99.5%
- 支持多设备数据同步
- 支持数据备份和恢复

## 3. 数据模型

### 3.1 主要数据实体

#### 用户（User）（新增）
```
{
  id: string,              // 唯一标识（用户ID）
  wechatOpenid: string,    // 微信OpenID
  wechatUnionid: string,   // 微信UnionID
  nickname: string,        // 用户昵称
  avatar: string,          // 用户头像URL
  phone: string,           // 手机号（可选）
  email: string,           // 邮箱（可选）
  status: string,          // 用户状态（active/banned）
  createdAt: timestamp,    // 注册时间
  updatedAt: timestamp     // 更新时间
}
```

#### 菜肴（Dish）
```
{
  id: string,              // 唯一标识
  userId: string,          // 创建者ID
  name: string,            // 菜名
  description: string,     // 描述
  difficulty: string,      // 难度（简单/中等/困难）
  cookingTime: number,     // 烹饪时间（分钟）
  servings: number,        // 份量
  image: string,           // 菜肴图片URL
  tags: string[],          // 标签列表
  ingredients: Ingredient[],  // 食材列表
  steps: CookingStep[],    // 制作步骤
  rating: number,          // 平均评分（1-5）
  ratingCount: number,     // 评分数量
  isFavorite: boolean,     // 当前用户是否收藏（前端）
  viewCount: number,       // 浏览次数（新增）
  likeCount: number,       // 点赞次数（新增）
  createdAt: timestamp,    // 创建时间
  updatedAt: timestamp     // 更新时间
}
```

#### 评论（Comment）（新增）
```
{
  id: string,
  dishId: string,          // 所属菜肴ID
  userId: string,          // 评论者ID
  userName: string,        // 评论者昵称
  userAvatar: string,      // 评论者头像
  content: string,         // 评论内容
  rating: number,          // 评分（1-5星）
  createdAt: timestamp,    // 评论时间
  updatedAt: timestamp     // 更新时间
}
```

#### 分享（Share）（新增）
```
{
  id: string,
  dishId: string,          // 被分享的菜肴ID
  userId: string,          // 分享者ID
  shareType: string,       // 分享类型（wechat_friend/wechat_group/qrcode）
  createdAt: timestamp     // 分享时间
}
```

#### 收藏（Favorite）（新增）
```
{
  id: string,
  userId: string,          // 用户ID
  dishId: string,          // 菜肴ID
  createdAt: timestamp    // 收藏时间
}
```

#### 关注（Follow）（新增）
```
{
  id: string,
  userId: string,          // 关注者ID
  followerId: string,      // 被关注者ID
  createdAt: timestamp    // 关注时间
}
```

#### 菜单（Menu）（新增）
```
{
  id: string,              // 唯一标识
  userId: string,          // 创建者ID
  name: string,            // 菜单名称
  description: string,     // 菜单描述
  tags: string[],          // 菜单标签（如：周末聚餐、家常菜等）
  coverImage: string,      // 菜单封面图片URL
  dishes: {                // 菜单中的菜肴
    dishId: string,        // 菜肴ID
    servings: number,      // 份量
    notes: string          // 备注信息
  }[],
  isPublic: boolean,       // 是否公开分享
  shareCount: number,      // 被分享的次数
  createdAt: timestamp,    // 创建时间
  updatedAt: timestamp     // 更新时间
}
```

#### 购物清单（ShoppingList）（新增）
```
{
  id: string,              // 唯一标识
  userId: string,          // 用户ID
  menuId: string,          // 关联的菜单ID
  items: {                 // 食材项
    id: string,
    name: string,          // 食材名称
    quantity: number,      // 数量
    unit: string,          // 单位
    category: string,      // 分类（蔬菜、肉类、调料等）
    checked: boolean,      // 是否已购买
    notes: string          // 购买备注
  }[],
  estimatedCost: number,   // 预计费用
  createdAt: timestamp,    // 创建时间
  updatedAt: timestamp     // 更新时间
}
```

#### 食材（Ingredient）
```
{
  id: string,
  name: string,            // 食材名称
  quantity: number,        // 数量
  unit: string            // 单位（克、个、汤匙等）
}
```

#### 制作步骤（CookingStep）
```
{
  id: string,
  stepNumber: number,      // 步骤序号
  description: string,     // 步骤描述
  image?: string           // 步骤图片（可选）
}
```

## 4. 用户流程

### 4.1 用户注册和登录流程（新增）
```
打开小程序 → 微信授权登录 → 完善个人信息 → 进入首页
```

### 4.2 新增菜肴流程
```
首页 → 点击"添加菜肴" → 填写菜肴信息 → 添加食材 → 添加步骤 → 上传图片 → 提交保存
```

### 4.3 浏览和互动流程
```
首页列表 → 点击菜肴 → 查看详情（食材、步骤、评论） → 可收藏/分享/评论/评分
```

### 4.4 搜索菜肴流程
```
首页搜索框 → 输入菜名/标签 → 显示搜索结果 → 点击查看详情
```

### 4.5 评论和评分流程（新增）
```
菜肴详情页 → 点击"写评论" → 输入评分和评论内容 → 提交评论 → 显示在评论列表
```

### 4.6 分享菜肴流程（新增）
```
菜肴详情页 → 点击"分享" → 选择分享方式（朋友/朋友圈/二维码） → 分享成功提示
```

### 4.7 关注用户流程（新增）
```
浏览他人菜肴 → 点击用户信息 → 进入用户主页 → 点击"关注" → 关注成功，可查看其新发布的菜肴
```

### 4.8 个人中心流程
```
点击底部菜单"我的" → 查看个人信息和菜肴统计 → 可修改信息、查看收藏和粉丝

## 5. 页面结构

### 5.1 页面列表
- **首页（Home）**：菜肴列表、搜索、排序、筛选

- **菜单列表（Menus）**（新增）：显示所有菜单、创建菜单
- **菜单编辑（MenuEditor）**（新增）：编辑菜单内容、添加删除菜肴
- **点菜界面（QuickOrder）**（新增）：快速点菜推荐、菜肴搭配
- **购物清单（ShoppingList）**（新增）：显示购物清单、编辑购物清单
- **分类页（Category）**：按标签/难度分类浏览
- **用户主页（UserProfile）**（新增）：显示其他用户发布的菜肴、用户信息、关注/粉丝
- **评论编辑（CommentEditor）**（新增）：添加和编辑评论的页面
- **粉丝列表（Followers）**（新增）：显示粉丝和关注列表
- **个人中心（Profile）**：用户信息、统计、收藏、粉丝、设置

#### 首页（Home）

- **菜单筛选**：按创建时间、菜肴数排序

#### 菜单编辑（MenuEditor）（新增）

- **菜肴列表**：已添加的菜肴列表，显示菜名、图片、份量

- **菜肴操作**：删除、调整份量、拖拽排序

- **食材分类**：按蔬菜、肉类、调料等分类显示
- **食材项**：显示食材名称、数量、单位、勾选框
- **核对功能**：可勾选已购买的食材
- **费用估算**：显示预计购物费用
- **编辑功能**：添加/删除食材、修改数量
- **分享按钮**：分享清单给家人或朋友
- **导出打印**：支持清单导出和打印

## 6. 业务规则

### 6.1 数据规则
- 菜名必填，长度 1-50 字符
- 难度级别必选（简单/中等/困难）
- 烹饪时间必填，范围 1-300 分钟
- 至少需要一个食材
- 至少需要一个制作步骤
- 最多支持 5 个标签
- 用户昵称长度 2-20 字符
- 评论内容长度 1-500 字符
- 评分范围 1-5 星

### 6.2 操作规则
- 删除菜肴需要二次确认
- 菜肴修改实时保存
- 搜索支持模糊匹配菜名和标签
- 同一用户同一菜肴只能评论一次（可编辑或删除）
- 用户可修改或删除自己的评论
- 关注功能不可关注自己
- 分享成功后分享数+1，浏览数据实时更新

### 6.3 权限规则
- 用户必须登录后才能添加菜肴、评论、分享、收藏
- 用户只能编辑/删除自己的菜肴和评论
- 用户可查看任何公开的菜肴和评论
- 用户可屏蔽/举报不当评论（可选）

### 6.4 隐私规则
- 用户默认菜肴为私密（仅自己可见），可选择分享给朋友或公开
- 用户可设置隐私等级：仅自己可见、仅关注者可见、公开
- 用户可禁用评论功能

### 6.5 通知规则（扩展）
- 用户发布新菜肴时，通知粉丝
- 菜肴被评论时，通知原创作者
- 用户被关注时，发送通知提醒

### 6.6 菜单和点菜规则（新增）
- 每个用户最多创建 50 个菜单
- 菜单名长度 1-30 字符
- 菜单最少包含 1 道菜，最多 20 道菜
- 菜肴份量范围：0.5-10 份
- 购物清单自动合并相同食材，相同单位的数量相加
- 菜单可以基于已有菜单复制创建，快速生成新菜单
- 菜单分享后，他人可以查看和复制菜单
- 购物清单数据仅在本地存储，根据菜单动态生成

## 8. 成功指标

- 用户能快速添加和查找菜肴
- 列表加载时间 < 300ms
- API 响应时间 < 1s
- 搜索响应时间 < 500ms

### 8.2 参与度指标
- 日均使用时长 > 5 分钟（上升）
- 用户保留率 > 60%（7天）
- 用户保留率 > 40%（30天）
- 日活跃用户数（DAU）
- 月活跃用户数（MAU）

- 平均每个菜肴被评论数 > 2

- 菜肴总数增长率

- 用户推荐率（朋友圈分享数）
